# 一、Cosmos 3：把世界模型 × 动作合一

如果说 SmolVLA 只有①策略、DIAMOND 只有②世界模型，那 **NVIDIA Cosmos 3**（2026-06 发布）就是当下把它们往一起捏得最狠的一个。它是一个面向 Physical AI 的**双塔 Mixture-of-Transformers**，基于 Qwen3-VL 权重：

<img src="images/smolvla_two_tower.svg" alt="双塔结构" style="width:100%;max-width:760px;display:block;margin:0 auto;">
<figcaption style="text-align:center;color:var(--secondary,#8B95A8);font-size:.9em;margin-top:.4rem;">双塔 Mixture-of-Transformers 的思路我们在第一篇 SmolVLA 已经见过；Cosmos 3 把它放大，并让两塔分别负责"推理"与"生成"。</figcaption>
</figure>

- **Reasoner 塔**：一个 VLM（Qwen3-VL 类），负责**物理推理**——理解画面里的物理。
- **Generator 塔**：一个扩散模型，**同时生成未来观测（世界生成）和动作序列（动作生成）**。

NVIDIA 给它起了个名字叫 **"World Action Model"**——把"世界模拟器 + 世界-动作模型"统进一个模型，一举统一 **物理推理 + 世界生成 + 动作生成**。

放回我们用了三篇的**三零件框架**里看：

<figure style="margin:1.4rem 0;">
<img src="images/planning_stack.svg" alt="规划三零件框架" style="width:100%;max-width:760px;display:block;margin:0 auto;">

- **SmolVLA**：①策略 ✅
- **DIAMOND**：②世界模型 ✅（但在 CSGO 域）
- **Cosmos 3**：①②统一，③部分 —— **最接近闭环的那个**。

变体上，Cosmos 3 有 **Nano（16B）**、**Super（64B）**、以及计划中的 **Edge（4B）**。

# 二、两堵墙：为什么它进不了机器人

看着很美。那把它下下来、装进机器人，行不行？我认为不能——挡在前面的是两堵**正交**的硬墙。

## 墙一：能耗

一个要**上身、实时**的具身智能体，功耗预算大概在 **≤100W** 量级。而一个要生成 720p 视频未来的统一大扩散模型，功耗远不止于此。

最有力的反证来自我们自己的脑子：**人脑用大约 20W，就实现了通用具身智能。** 它根本不需要"先在脑海里渲染出一段高清视频"才能决定下一步怎么动。这说明：**为了行动而生成像素级未来，在能耗上就是走不通的。**

## 墙二：延迟（一堵独立的墙）

就算你把能耗放开，还有第二堵墙：**实时控制必须在毫秒级出手。** 一个巨型扩散模型跑一次要几百毫秒甚至数秒——**这跟功耗无关，纯粹是太慢。**

所以即使能耗不设限，统一大模型也进不了实时控制的内循环。这两堵墙是**正交**的：一堵管"耗多少电"，一堵管"来不来得及"，各自都能单独把统一大像素模型挡在门外。

# 三、大脑给的答案：潜空间预测，不渲染像素

有意思的是，人脑其实**三个零件都齐**：策略（运动皮层）、世界模型（预测编码、小脑前向模型、海马 replay）、价值（多巴胺 RL）。它并没有绕开世界建模。

但**它的世界模型是潜空间的、稀疏的、可复用的——绝不渲染像素级的未来**，而且只用 20W。你伸手去拿杯子，大脑预测的是"手会到那儿、杯子会被握住"这种抽象后果，而不是先在视网膜上画出一帧未来的高清图像。

这就把方向指出来了：

> 未来高效的具身 AI，更像是**把潜空间的预测能力内化进策略本身**，而不是像 Cosmos 那样"生成一段视频再据此行动"。

世界模型的价值没有被否定——**否定的只是"以生成像素未来为形态"的世界模型上身**。

# 四、分层：云端工厂，边缘大脑

把两堵墙和大脑的启示合起来，一个清晰的分工浮现出来。这不是"统一大模型" vs "高效小策略"的**路线之争**，而是**分层**：

**☁️ 云端（能耗放开、离线、高价值）**
统一大世界模型 = **工厂 / 教师 / 模拟器**。它在这里最有价值：
- 生成训练数据、做 sim2real
- 造出罕见、危险的场景来测试策略
- 训练并**蒸馏（distill）**出要上身的那个策略

**Cosmos 3 就该待在这儿。**

**🤖 边缘（~20–100W、实时）**
真正部署上身的"大脑" = **高效的预测策略**——把潜空间预测内化、反应式地行动，由云端那个大模型**蒸馏**而来。

一句话收束这个反直觉的结论：

> **统一大模型不是部署在机器人身上的脑子，而是造那个脑子的工厂。** 它对世界的理解，最终要被蒸馏成一个 ~20W 的东西，才能真正上身。

Cosmos 代表的"**理解世界才能行动**"是对的方向；但它赌的"**生成像素未来**"这一形态，注定退居云端。

# 系列收束

四篇走下来，这条线其实很清楚：

1. **[(1) SmolVLA](/posts/smolvla-intro/)**：一个纯模仿策略——只有①，靠不断重看绕开建模。
2. **[(2) 可解释性](/posts/smolvla-interpretability/)**：撬开黑盒，看到它对场景的伪相关依赖，也撞上 best-of-N"只能挑运气"的天花板。
3. **[(3) DIAMOND](/posts/world-model-diamond/)**：世界模型登场，②补上了——但在另一个世界里，环没真闭。
4. **(4) Cosmos 与两堵墙**：统一是可能的，但能耗与延迟决定了它的位置在云端工厂，而非机器人身上。

最后留一个我自己也没有答案的**开放问题**：要让模仿学习出来的 VLA 真正"吃进未来"，到底走哪条路——

- **A. predictive-conditioning**：让策略直接条件于"想象出的未来"，但要重训 VLA；
- **B. MPC / Dreamer**：用世界模型推演候选、甚至在"梦里"训练，而 VLA 不改。

哪条更适合机器人操作，我还不知道。但有一点越来越确定：

> **未来属于「会预测的策略」；而世界模型的意义，是帮我们搞清楚——那个「预测」到底该长什么样。**
